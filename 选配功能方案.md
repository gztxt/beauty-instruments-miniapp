# 美容仪器小程序 — 选配功能添加方案

> 日期：2026-07-14
> 目标：在产品详情页增加「选配」功能，让用户为仪器选择**机箱样式 / 手柄样式 / 支架样式**等配件样式，实时预览所选配置，并在「立即咨询」时带上配置摘要。

---

## 一、需求定位（关键前提）

本项目是**展示型官网小程序**（采薇美容仪器），非电商：
- 详情页底部为「立即咨询 / 返回列表」，无购物车、无下单、无价格库存。
- 因此「选配」= **轻量配置选择器**：选样式 → 实时预览配置卡片 → 咨询时携带配置，而非电商 SKU 的「价格 × 库存」矩阵。

参考行业实现：微信生态标准是 Vant Weapp 的 `van-sku` 组件（规格维度 + 规格值 + 组合映射）。本方案借鉴其**多维度规格选择**的数据结构与交互，但去掉价格/库存，简化为「样式选择 + 配置汇总」。

## 二、数据结构设计

在 `data/products.js` 每个产品对象中，新增**可选**字段 `options`（数组，无此字段的产品不显示选配区，向后兼容）：

```javascript
// 产品对象新增字段（示例，加到需要选配的仪器上）
options: [
  {
    key: 'chassis',          // 维度唯一标识
    name: '机箱样式',         // 维度显示名
    required: true,          // 是否必选
    values: [
      { id: 'c1', label: '标准铝合金', desc: '阳极氧化银灰', icon: '🔲' },
      { id: 'c2', label: '高光烤漆白', desc: '钢琴烤漆白色', icon: '⬜' },
      { id: 'c3', label: '碳纤纹黑',   desc: '哑光碳纤维纹', icon: '⬛' }
    ]
  },
  {
    key: 'handle',
    name: '手柄样式',
    required: true,
    values: [
      { id: 'h1', label: '直柄经典', desc: '直握人体工学', icon: '🖊️' },
      { id: 'h2', label: '弯柄贴面', desc: '弧形贴合面部', icon: '🪝' },
      { id: 'h3', label: '双头手柄', desc: '一机双探头',   icon: '🔱' }
    ]
  },
  {
    key: 'bracket',
    name: '支架样式',
    required: false,
    values: [
      { id: 'b0', label: '不需要',   desc: '仅主机',       icon: '🚫' },

```

> 每个 `value` 有稳定 `id`（供咨询时回传/追踪），`label` 展示名，`desc` 副说明，`icon` emoji（沿用项目现有 features/scenes 的 emoji 风格，零图片资源成本）。

`data/products.js` 同时新增一个工具函数，给未配置 `options` 的旧产品返回空数组，保证详情页安全渲染：

```javascript
function getProductOptions(id) {
  const p = findProductById(id);
  return (p && Array.isArray(p.options)) ? p.options : [];
}
// 加入 module.exports
```

## 三、组件设计：`components/option-selector`

新增一个可复用自定义组件（沿用现有 `components/section-header` 的目录规范：js/json/wxml/wxss 四件套）。

**职责**：接收 `options` 数组，渲染每个维度的样式选择卡片；用户点击某样式即高亮选中；对外 `triggerEvent('change', {selection, summary})` 上报当前选择。

**Properties**：
- `options: Array` — 选配维度数据

**Data / 内部状态**：
- `selected: {}` — `{ [key]: valueId }` 记录每维度已选值；`required` 维度默认选中第一个，非必选默认不选。

**关键方法**：
- `initSelection()`：`observers` 监听 options 变化，初始化默认选中值。
- `onSelect(e)`：读取 `data-key` / `data-id`，更新 `selected`，重算 `summary`（`[{name, label}]`），`triggerEvent('change', ...)`。
- `buildSummary()`：把 `selected` 映射为可读的「维度名：样式名」列表，供预览卡片和咨询使用。

**WXML 结构**（每个维度一组）：
```
维度标题（含"必选"红点）
  → 横向排列的样式卡片（icon + label + desc），选中态描边高亮 + 勾选角标
配置预览卡片：汇总当前所有选择「机箱样式：碳纤纹黑 / 手柄样式：弯柄贴面 / 支架样式：桌面立式」
```

**WXSS**：复用项目现有 `card` 卡片风格与品牌绿 `#2d5016`（选中态描边/文字用品牌色），保证视觉统一。

## 四、页面改动：`pages/product-detail`

1. **product-detail.json**：`usingComponents` 注册 `"option-selector": "/components/option-selector/option-selector"`。
2. **product-detail.js**：
   - `onLoad` 取产品后，`this.setData({ options: getProductOptions(id) })`。
   - 新增 `data.optionSelection`（当前选择）与 `data.optionSummary`（摘要文本）。
   - 新增 `onOptionChange(e)`：保存 `e.detail.selection / summary`。
   - 改造 `goToContact`：跳转前把 `optionSummary` 写入全局或缓存（`wx.setStorageSync('inquiryConfig', {...})`），让「关于我们/咨询」页可读取展示（或至少 `console`/toast 提示"已记录您的选配"）。
3. **product-detail.wxml**：在「产品参数」与「产品特点」之间插入选配区块（仅当 `options.length > 0` 时显示）：
   ```
   <view wx:if="{{options.length}}" class="container section">
     <view class="section-title">选购配置</view>
     <view class="section-divider"></view>
     <option-selector options="{{options}}" bind:change="onOptionChange" />
   </view>
   ```
4. **product-detail.wxss**：新增选配区少量样式（大部分复用组件内样式）。

## 五、数据落地范围（第一批）

先给 **2~3 款代表性仪器**加 `options`（如 id:1 多功能美容仪、id:2 射频美容仪、id:3 射频紧肤仪），验证功能后再逐步补全。其余产品无 `options` → 详情页自动隐藏选配区，零副作用。

## 六、测试方案

沿用项目现有 Jest + `pages/about/__tests__` 的测试规范，新增：
- `data/products.test.js`：`getProductOptions` 对有/无 options 的产品分别返回数组/空数组。
- `components/option-selector/__tests__/option-selector.test.js`：
  - 初始化时 required 维度默认选中首项、非必选不选中。
  - `onSelect` 后 selected 更新且 triggerEvent 携带正确 summary。
  - buildSummary 输出格式正确。
- （可选）`product-detail` 页 onOptionChange 保存逻辑。

## 七、验收标准

1. 详情页有 options 的产品显示「选购配置」区，无 options 的产品不显示。
2. 点击样式卡片高亮切换，配置预览实时更新。
3. required 维度有默认选中，用户可改。
4. 「立即咨询」携带当前选配摘要（存入 storage 或提示）。
5. 新增单测全绿，`npm test` 通过。
6. 视觉风格（卡片/品牌绿/emoji）与现有页面一致。

## 八、三工具联动执行拆解

用 `pipeline.py` 分批喂给三模型（claude 规划 → codex 改码 → Hermes 测试/评审）：
- **Round 1**：`data/products.js`（加 options 字段 + getProductOptions + export） — 单文件串行。
- **Round 2**：`components/option-selector/*`（4 件套新组件） — 多文件，走 `--parallel`。
- **Round 3**：`pages/product-detail/*`（json/js/wxml/wxss 接入） — 多文件，走 `--parallel`。
- **Round 4**：测试文件生成 + `npm test` 验证 + 评审。
- 每轮 `--dashboard` 生成可视化报告，人工会签后进入下一轮。

> 备份先行、Codex 不碰生产库、Hermes 只跑不改、人会签、缓存回退 —— 遵循 pipeline.py 既有安全门禁。
